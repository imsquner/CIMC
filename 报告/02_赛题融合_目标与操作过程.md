## 1. 工程总目标（结合赛题）

### 目标 1：构建“采样-判定-显示-存储-审计”闭环系统
操作过程：
1. 上电后初始化时钟、GPIO、串口、Flash、RTC、ADC、OLED、TF 卡。
2. 进入协作式调度循环，周期执行按键、采样、显示、串口解析、LED 状态等任务。
3. 采样值按 ratio 换算，和 limit 比较后产生普通记录或超限记录。
4. 根据 hide/unhide 模式切换明文/编码输出，并写入对应目录。

代码落点：
- 主流程：User/main.c
- 初始化：APP/system_init.c
- 调度：APP/scheduler.c
- 采样与输出：APP/adc_app.c
- 串口命令：APP/uart_app.c
- 存储与日志：APP/sdcard_app.c

融合结论：已实现主闭环，具备赛题主线可演示能力。

### 目标 2：完成赛题要求的人机交互链路
操作过程：
1. 串口输入命令进入对应状态机（RTC Config、ratio、limit、start/stop、hide/unhide 等）。
2. 按键实现采样启停与周期切换（KEY1/KEY2/KEY3/KEY4）。
3. OLED 在 idle/运行态显示关键状态，LED1/LED2指示采样与超限。

代码落点：
- 串口：APP/uart_app.c
- 按键：APP/key_app.c
- OLED：APP/oled_app.c
- LED：APP/led_app.c

融合结论：交互能力完整；但部分参数校验和显示格式细节与赛题严格口径有差异。

## 2. 赛题逐条融合（目标+操作过程）

### 2.1 系统自检（test）
目标：输入 test 后完成 Flash/TF/时间状态回显。

操作过程：
1. 串口解析到 test。
2. 调用 system_test()。
3. 执行 disk_initialize(0)、spi_flash_read_id()、sd_card_capacity_get()。
4. 串口打印通过/失败与容量、时间信息，并写日志。

代码落点：APP/uart_app.c、APP/str_app.c。
融合判断：已实现（满足 2025 赛题主要求）。

### 2.2 RTC 设置与读取（RTC Config / RTC now）
目标：可设置当前时间并查询当前时间。

操作过程：
1. 发送 RTC Config，进入 rtc_set_flag。
2. 下一条时间字符串触发 rtc_reset(year,month,day,hour,minute,second)。
3. 发送 RTC now 回显当前时间。

代码落点：APP/uart_app.c、APP/str_app.c、APP/rtc_app.c。
融合判断：部分实现。
差异：时间解析目前写死了 "2025-06" 前缀判断；不满足赛题“格式更宽松”要求。

### 2.3 配置读取（conf + config.ini）
目标：从 TF 根目录读取 Ratio/Limit。

操作过程：
1. 发送 conf。
2. 打开 config.ini。
3. 解析 [Ratio] Ch0 和 [Limit] Ch0。
4. 输出 ratio/limit。

代码落点：APP/uart_app.c、APP/str_app.c。
融合判断：已实现。
差异：读取后未自动写入 Flash（赛题文字强调“更新至 Flash”流程）。

### 2.4 变比设置（ratio）
目标：查询并设置 ratio，校验有效范围。

操作过程：
1. ratio 命令进入 ratio_set_flag。
2. 下一条浮点文本输入进入 set_Ratio()。
3. 合法则更新 ratio_data，非法回显 invalid。

代码落点：APP/uart_app.c、APP/str_app.c。
融合判断：部分实现。
差异：边界判断近似为 0.9~100.1，与赛题 0~100 不完全一致。

### 2.5 阈值设置（limit）
目标：查询并设置告警阈值。

操作过程：
1. limit 命令进入 limit_set_flag。
2. 下一条浮点输入进入 set_Limit()。
3. 合法更新，非法不变。

代码落点：APP/uart_app.c、APP/str_app.c。
融合判断：基本实现。
差异：赛题流程中 0~200 与文档中 0~500 有版本差异；当前实现 0~500.1。

### 2.6 参数持久化（config save/read）
目标：变比/阈值掉电保存并可读取。

操作过程：
1. config save 将设备 ID + ratio + limit 写入 SPI Flash 地址 0/30/34。
2. config read 从地址 30/34 回读。

代码落点：APP/str_app.c、APP/SPI_FLASH.c。
融合判断：已实现。
差异：赛题测试流程中有“limit 不应持久化”场景，你们当前保存了 limit。

### 2.7 采样启停与周期切换（start/stop + KEY）
目标：串口和按键均可控制采样，支持 5/10/15s 切换。

操作过程：
1. start 或 KEY1 置 sp.led_flash=1，进入采样态。
2. stop 或 KEY1 再次触发置 0，回 idle。
3. KEY2/KEY3/KEY4 写 sp.sample_cycle=5/10/15。
4. 对应 print_adc_5s/10s/15s 任务在调度器中输出。

代码落点：APP/uart_app.c、APP/key_app.c、APP/adc_app.c、APP/scheduler.c。
融合判断：部分实现。
差异：print_adc_5s 在调度器周期为 500ms（应接近 5000ms）。

### 2.8 超限提示（LED2 + 串口）
目标：超过 limit 时亮 LED2 并输出 OverLimit。

操作过程：
1. adc_proc 刷新 sp.adc_vlot。
2. print_adc_xxs 比较 limit_data < adc_vlot。
3. 超限写 overLimit 文件并打印 OverLimit 文本。
4. led_proc 控制 LED2 状态。

代码落点：APP/adc_app.c、APP/led_app.c、APP/sdcard_app.c。
融合判断：已实现。

### 2.9 数据隐藏编码（hide/unhide）
目标：按赛题规则输出时间戳HEX+电压HEX。

操作过程：
1. hide 置 encrypt_flag=1，采样输出改为 hide_to_unix()。
2. hide_to_unix 组包 Unix 时间戳和电压整数/小数编码。
3. 超限时末尾追加 * 标记。
4. unhide 恢复明文输出。

代码落点：APP/uart_app.c、APP/str_app.c。
融合判断：已实现（比赛演示版）。
差异：时区处理和浮点精度细节建议再对齐官方校验工具。

### 2.10 TF 卡分目录存储
目标：sample/overLimit/log/hideData 四类数据按规则分文件。

操作过程：
1. sdcard_init 创建目录并准备 log 文件。
2. sample_proc/overlimit_proc/hidedata_proc 每 10 条切文件。
3. log_write 记录关键操作。
4. log_id 通过 Flash 地址 0x2000 持久化自增。

代码落点：APP/sdcard_app.c。
融合判断：已实现，且与赛题目录结构高度一致。

## 5. 你们工程可直接用于答辩的“目标+操作过程”版陈述

目标：
- 做一个面向工业现场的电压采集终端，支持参数配置、超限告警、分级存储和操作审计。

操作过程：
1. 上电初始化并写入系统日志。
2. 通过串口完成 RTC、ratio、limit、conf、config save/read 等参数动作。
3. start 后周期采样，按键可动态改周期。
4. 超限时 LED2 告警并写 overLimit。
5. hide 模式下输出并存储编码数据，unhide 恢复。
6. stop 后系统回 idle，全部记录保留在 TF 卡目录与 log 中。
